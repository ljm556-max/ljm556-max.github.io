2、context参数说明·

context 是一个普通的 JavaScript 对象，它暴露三个组件的 property：context.attrs(非响应式对象)、context.slots（插槽）、context.emit（方法）

3、执行 setup 时，组件实例尚未被创建。因此，你只能访问以下 property：

props、attrs、slots、emit
换句话说，你将无法访问以下组件选项：
data、computed、methods

5、this指向

官网说明：在 setup() 内部，this 不会是该活跃实例的引用，因为 setup() 是在解析其它组件选项之前被调用的，所以 setup() 内部的 this 的行为与其它选项中的 this 完全不同。这在和其它选项式 API 一起使用 setup() 时可能会导致混淆。

自己console.log测试后，setup里的this指向一般是undefined ，除非触发的时候有特定的指向。
