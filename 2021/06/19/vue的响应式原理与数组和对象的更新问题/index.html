vue的响应式原理与数组和对象的更新问题

1、响应式原理

1.1 原理

当你把一个普通的 JavaScript 对象传入 Vue 实例作为 data 选项，Vue 将遍历此对象所有的 property，并使用 Object.defineProperty 把这些 property 全部转为 getter/setter。Object.defineProperty 是 ES5 中一个无法 shim 的特性，这也就是 Vue 不支持 IE8 以及更低版本浏览器的原因。

这些 getter/setter 对用户来说是不可见的，但是在内部它们让 Vue 能够追踪依赖，在 property 被访问和修改时通知变更。这里需要注意的是不同浏览器在控制台打印数据对象时对 getter/setter 的格式化并不同，所以建议安装 vue-devtools 来获取对检查数据更加友好的用户界面。

每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把“接触”过的数据 property 记录为依赖。之后当依赖项的 setter 触发时，会通知 watcher，从而使它关联的组件重新渲染。

1.2 简单理解

data里面的数据都会被变成setter和getter, 然后watcher会监听这些变化！当data里面的数据被访问的时候回触发get函数！当data里面的数据被修改的时候，会触发set函数！ 修改数据，且进行虚拟DOM树比较，重新渲染模板内容！

2、数组与对象的更新问题

2.1 问题

  • data里面的某个数据是对象的时候：
    • 我们给他添加新的属性，页面不会响应。 this.person.sex 页面不会变化！
    • 删除他的某个属性的时候，页面也不会响应 delete this.person.name;
  • data里面的某个数据是数组的时候：
    • 通过下标的方式添加新值，修改值的时候，页面不响应！

2.2 原因

  • 上述这些方式添加的属性，修改数组， 都没有产生新的setter和getter。 vue 无法监听其变化！

2.3解决

  • 对象，数组
    • this.$set(对象/数组，键,值) Vue.set(对象/数组，键,值) 给对象增加新属性、给数组增加属性 都可以响应！
    • this.$delete(对象,键) 删除对象里面的某个属性，页面也响应！
  • 数组
  • 数组的修改推荐使用 api。【splice,push,shift,unshift,等等】这些api方法，vue已经做了响应式处理！
